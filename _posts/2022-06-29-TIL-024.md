## 사이드프로젝트 프론트엔드 기술선택
 
* react
  - 속도, 커리어, 경험  
* next.js
  - React 라이브라리의 프레임워크  
  - SSR를 위해서 실무에서 많이 사용한다.
  - 페이지 기반 라우팅 시스템

* redux
  - store 사용 (일종의 전역변수 느낌)
  - 자식 컴포넌트간의 다이렉트 데이터 전송이 불가능하므로 컴포넌트 수가 늘어나면 관리가 어렵다. 그래서 Redux를 사용
  - 많은 프로젝트에서 사용하므로 사용해보고 싶음

* Axios 
  - 백엔드가 Node라면 사용해보자
  - http 비동기 통신 라이브러리

* Styled component
  - JS 코드 내에 CSS 작성
